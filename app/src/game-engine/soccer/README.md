# Field League soccer module

## Integration contract

`SoccerScene` depends only on the typed `SoccerBridge` contract in `types.ts`.
Instantiate either:

- `createLocalSoccerBridge(...)` for deterministic 3v3 or 5v5 solo play; or
- an application adapter that receives authoritative `SoccerMatchSnapshot`
  objects and maps `MOVE`, `KICK`, `SELECT_TEAM`, and `RESTART` commands to the
  multiplayer transport.

Add `new SoccerScene(bridge)` to a Phaser configuration and start the
`SoccerScene` key. The scene owns presentation, camera, keyboard/touch input,
animation, and feedback. The bridge owns game truth. External snapshots must
use monotonically increasing revisions. The optional `advance(deltaMs)` hook is
reserved for local simulation and should be omitted by network bridges.

`SoccerSetup` emits `{ team, format, teamSize }`. `format: "quick"` maps to the
local bridge; `format: "crew"` is the multiplayer matchmaking handoff.

## Rules and simulation

- Regulation time: 180 seconds, adaptable from authoritative snapshot time.
- Field: 2400 x 1360 world units with 440-unit goal mouths.
- Fixed 60 Hz solo simulation with frame accumulation.
- Bounded player velocity, sprint energy, kick cooldown, exponential ball drag,
  capped kick impulse, sideline/end-line restitution, and explicit goal-mouth
  detection.
- Formation-aware AI: keepers guard a constrained goal lane, the nearest field
  player pressures the ball, and supporting players occupy passing lanes.
- Goal pause, kickoff reset, final whistle, and replay flow are deterministic.

## Assets and rendering

No downloaded asset is required. The pitch, stands, crowd, nets, players, ball,
HUD, controls, shadows, ball trail, and confetti are generated by Phaser vector
graphics. This keeps the module license-clean and stylistically aligned with the
project's plum, coral, teal, cream, and warm-sky playset palette.

If audio is attached by the application, recommended event mappings are:

- `SoccerCommand.KICK` -> short layered boot/ball transient;
- transition to `goal` -> crowd swell and goal sting;
- transition to `ended` -> referee whistle and result sting; and
- ball/player contact -> throttled, surface-aware tap.

## Performance budget

The default 5v5 match renders 10 player containers, one ball container, one
static pitch, and fixed HUD objects. Transient ball trails and goal confetti
self-destruct after their tween. No per-frame textures or DOM nodes are created.
